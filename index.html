为什么会写这个博客

转眼已经是大四的学生了,做了很多的笔试题,看了很多的面经,发现自己基础不牢靠,所以萌生了重新学习一边java基础的想法,然后又到了毕业的时候,就用个博客记录自己的java之路2.0吧

java的数据类型

java有两种大的数据类型,分为基本类型和引用类型.基本数据类型有八种类型,引用数据类型有三种类型

基本数据类型

八种基本数据类型又细分为四种,整数类型,浮点数类型,字符型和布尔型

整数类型

byte

占有一个字节 可取值范围为:-2^7~2^7-1
例子 byte a = 100;
不赋值的话默认值为0
需要注意的是,byte类型最大支持的范围只到-128-127,所以如果取byte b = 130;编译器会报错

short

占有两个字节 可取值范围为:-2^15~2^15-1
例子 short a = 100;
不赋值的话默认值为0

int

最重要的整数类型,大部分定义的整数类型都是int型的变量.
占有四个字节 可取值范围为:-2^31~2^31-1
例子 int a = 100;
不赋值的话默认值为0
int类型也可以用来表示八进制和十六进制的数
例子
int b = 010;//输出结果为 8
int c = 017;//输出结果为 9
int d = 018;//报错
例子
int e = 0x11;//输出结果为17
int f = 0x19;//输出结果为25
int g = 0x1F;//输出结果为32

long

一般用于比较大的整数类型,取值范围较大
占有八个字节 可取值范围为 :-2^63~2^63-1
例子 long a = 100L;

浮点数类型

浮点数可以用来表示小数,但是他不能用来表示精确的值,比如货币,因为浮点数会失去一定的精度,一般在货币中使用BigDecimal对高精度的数进行精确计算.

float

单精度浮点数类型 占有四个字节
例子 float f = 100.0f;
不赋值的话默认值为0.0f

double

双精度浮点数类型 占有八个字节
和int一样,一般我们使用浮点数时大部分时间是使用double类型进行定义
例子 double d = 100.0
不赋值的话默认值为0.0d

字符类型

字符类型是一个单一的字符,可以用来存储任何字符,包括中文.所有的字符本质还是数字,char字符是通过unicode编码,对char字符进行运算时,因为有着unicode编码表,所以会将字符对应的数字进行相加
占有两个字节
例子
char c1 = ‘铁’;//输出结果为铁
sout((int)c1);//输出结果为38081
char c2 = ‘a’;//输出结果为a
sout((int)c2);//输出结果为97
sout(c1+c2);//输出结果为38187
char c3 = (char)(c1+c2);//输出结果为锢
char c4 = ‘\u0097’//输出结果为a
从例子中可以看出,每个中文字符都有对应的数字,英文字符和符号按照ASCII码表中的值为数字

布尔类型

布尔类型只表示一位信息,只有两个取值,分别是true和false
布尔类型只能用来作为一种标志来记录是非情况
例子 boolean flag = true;
不赋值的话默认值为false

类型转换

不同类型的数据可以进行转换

字符类型转换优先级

byte,short,char < int < long < float < double
一般是通过数据类型占有的字节大小判断优先级,而浮点数类型的优先级比整数类型高
在运算中,不同类型的数据先转换为同一数据类型,然后进行运算.

强制类型转换

数据类型从高到低转换时,需要进行强制类型转换,也就是俗称的强转.
具体方法为,在变量前加个括号
int a = 127;
byte b = (byte)a;//输出结果为 127
如果int的值过大,就会造成内存溢出的现象
int a = 129;
byte b = (byte)a;//输出结果为-128

自动类型转换

数据类型从低到高时,不需要我们进行类型转换,因为java自动帮我们进行了类型转换
int a = 128;
float = a ;//输出结果为128.0
运算时,java也会进行自动类型转换
例如 int与char进行运算就会将结果自动转换为int型的数据,如果数据不超界的话可以通过强转转换成相应char字符,这边具体就不举例了,大家可以自己在ide上试试.
而运算时应注意数据超界
当int与int进行运算后数据超界,使用long类型进行接收时,我们会发现long类型依旧是一个意义不明的数字
例子
int a = 10_0000_0000;
int b = 10;
int total = ab;//输出结果很明显会超界,10亿10已经超出了int类型的值所能表示的范围.
long total2 = ab;虽然我们使用了long类型进行存储,但是大家在运行代码时会发现,total2已经是total1的值,这是因为ab在运算时就已经超界,内存溢出了,哪怕是使用了long类型进行接收,结果依然是错误的,我们应该在运算时就进行强转
long total3 = a*(long)b;//输出结果为100_0000_0000;
这样java在运算时会自动的将数据转换为long类型进行运算,而数据超界的问题就解决了.
需要注意的是,不能对布尔类型进行类型转换
也不能将基本数据类型转换为引用数据类型
转换的时候应该注意内存溢出和精度问题
